# سامانه ریشه

افزونه یکپارچه مدیریت کسب‌وکار برای وردپرس و ووکامرس.

## پیش‌نیازها

- وردپرس 6.5 یا جدیدتر
- PHP 8.1 یا جدیدتر
- Composer 2 برای توسعه
- MySQL 8 یا MariaDB 10.6 و جدیدتر

## توسعه

```bash
composer install
composer lint
composer test
composer release:candidate
```

## بخش‌های پیاده‌سازی‌شده

- زیرساخت، ثبت رویداد، جلوگیری از ثبت تکراری، صف خروجی و Migration
- حسابداری چهارسطحی و اسناد غیرقابل‌تغییر
- انبار بچ‌محور، رزرو، انتقال و بهای تمام‌شده
- تولید، فرمول ساخت، ضایعات، دستمزد و سربار
- فروش چندکاناله، ارتباط با مشتریان، تخفیف، وفاداری و ووکامرس
- خزانه‌داری، لینک پرداخت، Callback امضاشده، تسویه و مغایرت‌گیری
- خرید، هزینه‌های جانبی، بدهی تأمین‌کننده و حساب‌های پرداختنی
- فروش سازمانی و امانی، عامل فروش، کمیسیون، اعتبار و تسویه
- لجستیک، استعلام، برچسب، رهگیری، هزینه و تسویه
- صورتحساب رسمی ایران، امضا، ارسال، استعلام، اصلاح، ابطال و برگشت
- مرکز عملیات، صف پایدار، تلاش مجدد، رخدادها و بررسی سلامت
- ابزارهای انتشار، Backup/Restore، بسته امضاشده و تست واقعی دیتابیس
- تحلیل رویدادمحور، هدف‌گذاری، انتساب منبع فروش، تاریخچه قیمت و هشدارها
- رابط گرافیکی کامل وردپرس برای تمام ماژول‌های پیاده‌سازی‌شده

نسخه `1.6.0` علاوه بر رابط مدیریت فارسی و راست‌چین، اتصال عملیاتی ووکامرس را اضافه می‌کند. شناسه‌های داخلی API و پایگاه‌داده برای حفظ سازگاری تغییر نکرده‌اند. نسخه سازگاری پایگاه‌داده `2026071925` است.

## اتصال کامل ووکامرس

در این نسخه «ریشه» مرجع موجودی است و ووکامرس کانال فروش محسوب می‌شود:

- ثبت و به‌روزرسانی سفارش‌های ووکامرس در فروش ریشه
- رزرو موجودی ریشه هنگام Checkout و قطعی‌کردن خروج پس از پرداخت
- آزادسازی رزرو سفارش لغوشده پیش از پرداخت
- برگشت موجودی اقلام مرجوعی فقط وقتی گزینه بازگرداندن به انبار در ووکامرس انتخاب شده باشد
- تغییر وضعیت سفارش ریشه به مرجوع‌شده و معکوس‌کردن سند حسابداری در مرجوعی کامل
- نگاشت خودکار محصول ساده و Variation بر اساس شناسه ووکامرس یا SKU
- ارسال موجودی قابل‌فروش ریشه به ووکامرس پس از رسید، تولید، انتقال، فروش، ضایعات یا اصلاح
- دریافت اصلاح دستی موجودی ووکامرس در ریشه، در صورت فعال‌بودن این گزینه
- مغایرت‌گیری زمان‌بندی‌شده و اجرای دستی Push/Pull از پنل تنظیمات
- جلوگیری از کسر و رزرو دوباره موجودی توسط ووکامرس، زیرا رزرو در ریشه انجام می‌شود
- ثبت خطا، آخرین اجرای همگام‌سازی و اجرای مجدد از طریق Action Scheduler یا WP-Cron

### راه‌اندازی

1. از «ریشه ← تنظیمات ← اتصال کامل ووکامرس» یک انبار ریشه را انتخاب کنید.
2. ابتدا «ورود و اتصال محصولات» را اجرا کنید.
3. جهت مرجع مغایرت را روی «ریشه» نگه دارید، مگر اینکه برای شروع موجودی ووکامرس منبع اولیه باشد.
4. یک‌بار «دریافت موجودی از ووکامرس» یا «ارسال موجودی ریشه به ووکامرس» را متناسب با منبع اولیه اجرا کنید.
5. اتصال، سفارش‌ها، موجودی و مرجوعی را فعال کنید و تنظیمات را ذخیره کنید.

قواعد توسعه در `AGENTS.md` و مستندات فنی در پوشه `docs/` قرار دارند.
